## **目录说明**
1. **`/config`**
   - 存放项目的主配置和子配置文件。
   - 示例文件：`config.go`。

2. **`/controllers`**
   - 处理请求的控制器层，接收用户请求并返回响应。
   - 示例文件：`user_controller.go`。

3. **`/models`**
   - 定义数据库表结构的模型层。
   - 示例文件：`user.go`。

4. **`/services`**
   - 实现核心业务逻辑的服务层。
   - 示例文件：`auth_service.go`。

5. **`/routes`**
   - 定义路由的模块化目录。
   - 示例文件：`user_routes.go`。

6. **`/middleware`**
   - 定义全局或模块化中间件。
   - 示例文件：`jwt_middleware.go`。

7. **`/database`**
   - 负责数据库连接、模型迁移与初始化。
   - 示例文件：`database.go`。

8. **`/logs`**
   - 存储日志文件，支持分割与管理。
   - 示例文件：自动生成的日志文件。

9. **`/storage`**
   - 存放文件上传的目标目录。
   - 示例文件：上传的用户文件。

10. **`/utils`**
    - 存放通用工具函数。
    - 示例文件：`utils.go`。

## **命名规则**

### **代码命名**
| **类别**           | **命名规则**             | **示例**            |
|-------------------|---------------------------|---------------------|
| 变量和属性        | 小写字母开头，驼峰命名   | `userName`          |
| 函数和方法        | 小写字母开头，驼峰命名   | `getUserByID()`     |
| 导出函数与结构体  | 大写字母开头，驼峰命名   | `GetUser()`         |
| 常量              | 全大写，单词间用下划线   | `DEFAULT_TIMEOUT`   |
| 接口              | 大写字母开头，带 `er` 后缀 | `Logger`            |
| 包名              | 小写英文字母，无特殊符号 | `auth`, `database`  |

---

### **数据库命名**
1. 表名：全小写，单数形式。
   - 示例：`user`, `role`, `log`。
2. 字段名：使用 `snake_case`（下划线命名）。
3. 主键名：统一为 `id`。
4. 外键名：统一为 `{表名}_id`。

## **标准化建议**
- **目录划分清晰：** 每个模块的职责单一，避免交叉管理。
- **代码易读性：** 使用一致的命名风格，保证代码可维护。
- **灵活扩展：** 按需新增模块目录，例如 `tasks`, `notifications`。
